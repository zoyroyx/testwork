## 🚀 Основная информация

* **Язык разработки:** Python 3.11+ (протестировано на Python 3.12.10)
* **Фреймворк:** FastAPI
* **База данных:** PostgreSQL (Asyncpg) + SQLAlchemy 2.0 (через асинхронные сессии)
* **Миграции:** Alembic
* **Контейнеризация:** Docker & Docker Compose
* **Тестирование:** Pytest + pytest-asyncio + aiosqlite (интеграционные тесты занимают **< 0.2 сек**)
* **Качество кода:** Конфигурация Ruff (Linter & Formatter) и Mypy (Type Checker) в `pyproject.toml`
* **CI/CD:** Автоматический конвейер сборки и проверки качества кода через GitHub Actions

## 🏗️ Метрики и Архитектура

* **Количество эндпоинтов:** 6 API-эндпоинтов (включая हेल्थ-чеки `/health` и `/ready`).
* **Количество таблиц в БД:** 2 (`approval_requests` — заявки, `approval_logs` — аудит-лог действий).
* **Слоистая структура:** `routers` ➔ `services` ➔ `repositories` (DAO).
* **Чистота кода:** В соответствии со строгими правилами проекта, все эндпоинты в роутерах занимают **не более 10 строк кода**, делегируя работу сервисному слою.

## 🛡️ Безопасность и Инженерия

1. **Multi-tenancy (Изоляция данных):** Каждый запрос к базе данных фильтруется по `workspace_id`. Доступ к чужому воркспейсу перехватывается на уровне проверки прав (HTTP 403) или скрывается (HTTP 404), предотвращая утечку существования данных.
2. **Идемпотентность:** Создание записей контролируется заголовком `Idempotency-Key`. Повторный клик отправляет клиенту статус `200 OK` с уже созданным объектом без дублирования в базе.
3. **Конечный автомат (State Machine):** Статус заявки движется только в одну сторону: `PENDING` ➔ `APPROVED` | `REJECTED` | `CANCELED`. Любая попытка изменить финальный статус отсекается с ошибкой `409 Conflict`.
4. **Аудит изменений:** Каждое изменение статуса пишется в таблицу `approval_logs` в рамках одной атомарной транзакции (Single Commit).
5. **Предотвращение N+1:** В слое репозиториев используется жадная загрузка связанных связей через `selectinload` для оптимизации количества запросов к БД.
6. **Защита PII:** Настроено безопасное логирование, исключающее вывод паролей, токенов, email-адресов или других персональных данных.

## ⚡ Интересные факты

* **0 Warnings:** Мы полностью избавились от 8 предупреждений об устаревании `datetime.utcnow()` в Python 3.12+, внедрив собственную наивную обертку над `datetime.now(timezone.utc)`.
* **Zero-Trust Input:** Входящие и исходящие JSON сериализуются через строгие схемы Pydantic с camelCase-алиасами для интеграции с фронтенд-клиентами.
* **Готовность к Event-Driven:** В сервисный слой встроен хук `publish_event`, спроектированный для реализации паттерна **Transactional Outbox** для бесконфликтной интеграции с брокерами сообщений (Kafka/RabbitMQ).
